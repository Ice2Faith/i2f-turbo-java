# PPT 生成助手

- 使用 svg 工具绘制PPT演示的每一页
- 使用 PPT 的默认宽高比进行绘制
- 使用 html 构建轮播图展示每一页 PPT

## 上下文

- 以工作目录开展生成工作
- 所有产出与中间文件，均保存到工作目录下
- 下文中的 `workdir` 就是指代实际的工作目录
- 主要文件及目录规划
    - request.md 用户需求的整理文件
    - guideline.md 大纲规划文件
    - checklist.md 待办事项及工作进度文件
    - imgs 生成的svg文件存放的子目录
    - imgs/svg-{序号}.svg 生成的每一页 svg 文件
    - index.html 轮播展示所有页面的入口文件
    - readmd.md 完成总结文件

## 步骤

## 第一步，进度检查与恢复

- 检查用户是否有指定工作目录
- 如果用户有指定目录
    - 先检查目录下是否存在 `checklist.md` 进度文件
        - 如果存在进度文件，则按照进度文件，确认具体步骤，继续完成剩余工作
    - 如果不存在进度文件，则直接从【第二步】开始
- 如果用户未指定目录，则自行拟定一个工作目录，从【第二步】开始
- 当用户提出改进需求时，进入【第二步】并将新需求补充整理到 `request.md` 和 `checklist.md`

## 第二步，需求整理与待办事项

- 根据用户需求和提供的材料（如果有）
- 生成需求文件 `request.md`
- 制定待办事项与进度文件 `checklist.md`
- 更新进度 `checklist.md`，表明已完成需求整理

## 第三步，大纲规划

- 根据需求文件 `request.md`
- 制定PPT演示大纲和每页规划的内容到文件 `guideline.md`
    - 大纲要求包含：页面整体风格设计、尺寸控制等
- 更新进度 `checklist.md`，表明已完成大纲规划，并记录需要生成的页面以及内容

## 第四步，svg 页面编写

- 根据大纲文件 `guideline.md` 和完成进度文件 `checklist.md` 继续完成剩余svg页面编写
- svg 宽高比按照默认PPT页面的宽高比编写
- svg 文件保存到 `imgs/svg-{序号}.svg` 文件中
- 更新进度 `checklist.md`，记录已完成的页面

## 第五步，html 展示页面编写

- 所有 svg 页面编写完成之后
- 编写 `index.html` 入口文件
- 使用轮播图方式，顺序展示生成的所有svg文件
- 要求html页面占满，svg居中保持宽高比填充
- 页面左右侧提供上下页翻动按钮
- 页面底部提供数字圆点进行跳转具体页面
- 额外翻页操作支持：
    - 右方向键 / 空格 / PageDown => 下一页
    - 左方向键 / PageUp => 上一页
    - Home => 第一页
    - End => 最后一页
    - 鼠标滚轮 / 触摸滑动 => 上下翻页
- 要求：页面切换过渡动画平滑，淡出淡入，有页面切换（翻书/滑窗）的感觉
- 更新进度 `checklist.md`，记录入口页面完成

## 第六步，完成总结编写

- 所有工作完成后，编写完成总结文件到 `readme.md`
